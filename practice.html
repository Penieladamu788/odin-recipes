INDEXES

Indexes are special lookup tables that the database search engine can use to speed up data retreival.
Simply put, an index is a pointer to data in a table. An index in a database is similiar to an index at the book/ bible.
Index are automatically created when primary key and unique constraints primary key and unique constraints are defines as table columns.

INDEX CLASSIFICATION

  1. Clustered and Non-Clustered Index
  2. Dense and Sparse Index
  3. Primary and Secondary Index
  • Clustered and Non-Clustered

    • Clustered index stored the data rows or view based on their key value. These are the columns included in the index definition.
    There can only be one clustered index per table, because the data rows themselves can be sorted in only one order.
    Non-clustered index has a structure seperate from data rows.
    A non-clustered index key values and each key value entry has a pointer to the data row that contains the key value. Its data rows are stored in an unordered structure called heap.

  • Dense and Sparse Index

    • A dense index in a database is a file with pairs of keys and pointers for every record in a file. Every key in this file is associated with a particular
    pointer to record in the sorted data file. An index record appears for every search key value and a pointer to the actual record.
    A sparse index is a file with pairs of keys and pointers for every block in the data file.
    Every key in this file is associated with a particualar pointer to the block to the sorted data file. In clustered indexes with duplicate keys, the sparse index points to the lowest search key in each block.
    In index, records are created only for some of the records.

  • Primary and Secondary Index

    • Primary indexcontains the key fields of the table and a pointer to non-key fields of the table. The primary key index is created automatically when the table is created in the database.
    Secondary index is used to index fields that are neither ording fields nor key fields (Here is no assurance to the file is organized on key field or primary key). When you create the file you must first specify two things abaout the primary index:

    1. The name of the primary index
    2. The primary index specification

DATABASE SECURITY

Database security refers to the collective measures used to protect and secure a database/database management software/system from illegitimate use and malicious threats and attacks.
It is a broad term that includes a multiple of processes, tools and methodologies that ensures security within a database environment.
Database security covers and enforces security on all aspect and components of database.
This includes:

  • Data stored in database
  • Database server
  • DBMS
  • Other database workflow applicaton

The three main objectives to consider when designing a secure database are;

  • Security
  • Integrity
  • Availability

Data Security

It is a practice of keeping data protected from corruption and unauthorized access.
The focus beheind data security is to ensure privacy while protecting personal or cooperate data.

Importance of Data Security
